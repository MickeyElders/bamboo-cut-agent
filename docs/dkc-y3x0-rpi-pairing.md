# DKC-Y3X0 Cutter Pairing

这份文档定义 `DKC-Y320 / DKC-Y360` 与树莓派之间的配对方案，目标是让 DKC 负责实时运动控制，树莓派负责 UI、流程协调和 Modbus 指令下发。

## 结论

可以基于现有 `.prt` 工程思路写出我们需要的 DKC 程序，但更稳妥的做法不是直接生成 `.prt` 二进制，而是明确：

- DKC 内部线程程序
- D/M 寄存器协议
- 树莓派侧 Modbus 驱动

仓库里的树莓派后端已经补上 `DKC-Y3X0 + RS485 Modbus` 通道，见：

- [modbus_rtu.py](/d:/github/bamboo-cut-agent/backend/app/modbus_rtu.py)
- [dkc_y3x0_cutter.py](/d:/github/bamboo-cut-agent/backend/app/dkc_y3x0_cutter.py)

## 适用目标

本方案只针对一根刀轴 Z 方向：

- `电机 1 / S1` 负责刀片上下运动
- 上安全位定义为 `0 mm`
- 下压方向定义为正方向
- DKC 内部负责真实位置闭环和软限位
- 树莓派只发“上升 / 下降 / 设零”请求

## DKC 参数设置

在 DKC 电机 1 参数里建议这样配置：

- 电机对象: `S1`
- 单位名称: `毫米`
- 每圈脉冲: 按驱动器细分填写
  - 例: `DM542 = 1600`
- 每圈距离: 按丝杆导程填写
  - 例: 丝杆导程 `8 mm` 就填 `8`
- 脉冲模式: `单脉冲`
- 位置寄存器: `D1`
- 速度寄存器: `D11`
- 负软限位: `0`
- 正软限位: 按刀轴最大允许下压距离填写
  - 例: `60`

可选保护：

- 负限位触发: `M20`
- 正限位触发: `M21`

说明：

- 说明书明确写了位置寄存器和速度寄存器是只读反馈寄存器，不是直接写入控制目标。
- 所以树莓派不应该直接“改 D1 让电机动”，而应该通过线程程序解释命令。

## D/M 协议

### 反馈寄存器

- `D1`: 电机 1 实时位置，单位 mm
- `D11`: 电机 1 实时速度

### 树莓派写入参数

- `D30`: 下压目标绝对坐标
- `D31`: 下压速度
- `D32`: 上升目标绝对坐标
- `D33`: 上升速度
- `D34`: 设零时写入的新坐标，通常为 `0`

默认建议：

- 位置类寄存器统一按 `mm * 1000` 传输
- 也就是 `1.234 mm -> 1234`
- 树莓派环境变量使用 `DKC_POSITION_SCALE=1000`

### 树莓派触发位

- `M1`: 下压请求
- `M2`: 上升请求
- `M3`: 设零请求

### DKC 返回状态位

- `M10`: 忙碌中
- `M11`: 完成脉冲
- `M12`: 故障/限位

### 可选限位位

- `M20`: 负限位触发
- `M21`: 正限位触发

## 位语义

DKC 里要注意一件事：

- `输出闭合` 会把 `M` 写成 `0`
- `输出断开` 会把 `M` 写成 `1`
- `输入闭合` 检测 `M=0`
- `输入断开` 检测 `M=1`

所以本方案约定：

- 树莓派通过 `M1/M2/M3` 发一个 `1` 脉冲作为请求
- DKC 线程里用 `输入断开` 等待请求
- 状态位 `M10/M11/M12` 也统一按 `1=激活` 理解

## 线程程序

下面的程序是建议版，可以直接照着在 YB3 里录。

### THR1 下压线程

用途：

- 等待树莓派发 `M1=1`
- 读取 `D30 / D31`
- 让 `S1` 绝对运动到下压目标

程序：

1. `输入断开` `M1` `0`
2. `输出闭合` `M1` `-`
3. `输出闭合` `M11` `-`
4. `输出断开` `M10` `-`
5. `速度` `S1` `D31`
6. `单机位移` `S1` `D30`
7. `输出闭合` `M10` `-`
8. `输出断开` `M11` `-`
9. `延时` `0.05` `-`
10. `输出闭合` `M11` `-`
11. `跳转` `1` `-`

### THR2 上升线程

用途：

- 等待树莓派发 `M2=1`
- 读取 `D32 / D33`
- 让 `S1` 绝对运动到上升目标

程序：

1. `输入断开` `M2` `0`
2. `输出闭合` `M2` `-`
3. `输出闭合` `M11` `-`
4. `输出断开` `M10` `-`
5. `速度` `S1` `D33`
6. `单机位移` `S1` `D32`
7. `输出闭合` `M10` `-`
8. `输出断开` `M11` `-`
9. `延时` `0.05` `-`
10. `输出闭合` `M11` `-`
11. `跳转` `1` `-`

### THR3 设零线程

用途：

- 等待树莓派发 `M3=1`
- 把当前电机坐标改成 `D34`

程序：

1. `输入断开` `M3` `0`
2. `输出闭合` `M3` `-`
3. `设置坐标` `S1` `D34`
4. `输出断开` `M11` `-`
5. `延时` `0.05` `-`
6. `输出闭合` `M11` `-`
7. `跳转` `1` `-`

## 基于当前 PRT 的改造方法

当前模板文件里已经有 `THR1 / THR2 / THR3`，但它们不是我们现在要的刀轴控制逻辑。

从 `prt` 中提取到的现状大致是：

- `THR1` 同时操作 `S1` 和 `S2`，并引用 `D30 / D31`
- `THR2` 主要在轮流切换 `OUT1-OUT5`
- `THR3` 只明显使用了 `M1` 和 `M10`

这说明：

- 这个工程文件结构可用
- 但线程逻辑是模板/测试逻辑，不是当前设备的正式逻辑

### 改造原则

- 保留 `THR1 / THR2 / THR3` 线程名，不新增线程名
- 直接把原线程内容覆盖掉
- `THR1` 改成下压线程
- `THR2` 改成上升线程
- `THR3` 改成设零线程

### THR1 替换稿

当前 `prt` 中的 `THR1` 有 16 行，建议直接清空后改成下面 11 行：

1. `输入断开` `M1` `0`
2. `输出闭合` `M1` `-`
3. `输出闭合` `M11` `-`
4. `输出断开` `M10` `-`
5. `输出闭合` `M12` `-`
6. `速度` `S1` `D31`
7. `单机位移` `S1` `D30`
8. `输出闭合` `M10` `-`
9. `输出断开` `M11` `-`
10. `延时` `0.05` `-`
11. `跳转` `1` `-`

处理方式：

- 原来涉及 `S2` 的行全部删掉
- 原来重复的 `D31` 行全部删掉
- 不要在这个线程里再操作 `S2`

### THR2 替换稿

当前 `prt` 中的 `THR2` 是一组 `OUT1-OUT5` 闪烁逻辑，建议整段覆盖成上升线程：

1. `输入断开` `M2` `0`
2. `输出闭合` `M2` `-`
3. `输出闭合` `M11` `-`
4. `输出断开` `M10` `-`
5. `输出闭合` `M12` `-`
6. `速度` `S1` `D33`
7. `单机位移` `S1` `D32`
8. `输出闭合` `M10` `-`
9. `输出断开` `M11` `-`
10. `延时` `0.05` `-`
11. `跳转` `1` `-`

处理方式：

- 原有的 `OUT1 ~ OUT5` 行全部删除
- 原有延时闪烁逻辑全部删除
- 这个线程只保留上升动作

### THR3 替换稿

当前 `prt` 中的 `THR3` 已经有一点 `M1 / M10` 的影子，但不完整，建议覆盖成设零线程：

1. `输入断开` `M3` `0`
2. `输出闭合` `M3` `-`
3. `输出闭合` `M11` `-`
4. `输出闭合` `M12` `-`
5. `设置坐标` `S1` `D34`
6. `输出断开` `M11` `-`
7. `延时` `0.05` `-`
8. `跳转` `1` `-`

处理方式：

- 不再使用 `M1` 作为设零触发
- `THR3` 只负责设零，不负责运动

### 主程序建议

如果这个模板的主程序仍然保留了启动其他线程或测试逻辑，建议改成最小初始化：

1. `输出闭合` `M10` `-`
2. `输出闭合` `M11` `-`
3. `输出闭合` `M12` `-`
4. `线程启动` `THR1` `-`
5. `线程启动` `THR2` `-`
6. `线程启动` `THR3` `-`
7. `跳转` `7` `-`

如果当前模板主程序没有显式主线程入口，也可以在运行设置中确保这三个线程会被启动。

### 页面部分建议保留和删除

建议保留：

- `D1` 的位置显示
- `D11` 的速度显示
- `D30 / D31` 这类寄存器输入框，如果你还想在 DKC 本地屏上调试
- `M10` 的状态显示

建议删除或忽略：

- 多轴调试区 `S2 ~ S6`
- `OUT1 ~ OUT5` 的流水灯测试区
- 与本机刀轴无关的输入输出演示组件

### 软限位建议

当前模板里看到：

- `SoftLimit:0`
- `SoftLimit2:0`

这表示软限位未启用。实际使用时至少要把 `S1` 改成：

- 负软限位 `0`
- 正软限位 `最大安全下压距离`

例如：

- 刀轴最多允许下压 `42 mm`
- 那就设置 `0 ~ 42`

### THR4 负限位监控线程

只有在你给电机 1 绑定了 `M20` 为负限位触发时才需要。

程序：

1. `输入闭合` `M20` `0`
2. `停转` `S1` `0`
3. `输出断开` `M12` `-`
4. `跳转` `1` `-`

### THR5 正限位监控线程

只有在你给电机 1 绑定了 `M21` 为正限位触发时才需要。

程序：

1. `输入闭合` `M21` `0`
2. `停转` `S1` `0`
3. `输出断开` `M12` `-`
4. `跳转` `1` `-`

## 上电初始化建议

主程序建议做最小初始化：

1. `输出闭合 M10`
2. `输出闭合 M11`
3. `输出闭合 M12`
4. `线程启动 THR1`
5. `线程启动 THR2`
6. `线程启动 THR3`
7. `线程启动 THR4`

## 树莓派如何配对

树莓派侧当前已经支持这些环境变量：

- `DKC_SERIAL_PORT`
- `DKC_BAUDRATE`
- `DKC_SLAVE_ID`
- `DKC_CUTTER_DOWN_M`
- `DKC_CUTTER_UP_M`
- `DKC_CUTTER_BUSY_M`
- `DKC_CUTTER_DONE_M`
- `DKC_CUTTER_FAULT_M`

当前后端已经能：

- 通过 `M1` 触发下压
- 通过 `M2` 触发上升
- 轮询 `M10 / M11 / M12`

还差一小步就能完全和这份 DKC 程序配齐：

- 写 `D30 / D31 / D32 / D33 / D34`
- 触发 `M3`
- 读取 `D1` 作为真实位置回读

## 推荐最终职责

### DKC 负责

- 电机脉冲输出
- 绝对运动
- 软限位
- 限位/故障保护
- 实时位置反馈

### 树莓派负责

- UI
- 模式切换
- 参数持久化
- 触发运动命令
- 读取状态并显示

## 为什么这样做

这是这套设备更稳的结构：

- 高实时的动作闭环留在 DKC
- 树莓派不直接做脉冲发生器
- UI 和流程控制不依赖 DKC 触摸屏操作
- 后续接云端时，云端只需要调树莓派暴露的能力接口
